MERRYLIGHT ERP - GO-LIVE RUNBOOK (hosted app as the team's primary)
====================================================================

STEP 1 - BACKEND FIRST (~10 min, do this before rollout)
  As the primary team app, it needs shared data across devices.
  Follow README-BACKEND.txt: create the free Supabase project, run
  setup-backend.sql, paste the URL + anon key into backend-config.js.
  Skipping this = every phone keeps its own separate data.

STEP 2 - HOST (~2 min)
  Drag this whole folder onto https://app.netlify.com/drop
  Note the URL it gives you. In Netlify settings you can rename it
  (e.g. merrylight.netlify.app). This URL is now "the app".

STEP 3 - FIRST SIGN-IN (you, once)
  Open the URL. Sign in: admin / 1234. Immediately:
  - Users page -> Change my PIN
  - Users page -> add each team member with the right role
    (Admin / Data entry / View only)

STEP 4 - ROLL OUT TO THE TEAM
  Send the URL. Each person:
  - Android: open in Chrome -> menu -> Install app
  - iPhone: open in SAFARI -> Share -> Add to Home Screen
  - Desktop: just bookmark it (Chrome also offers Install)
  Then they sign in with the account you created for them.

ONGOING - HOW UPDATES WORK FROM NOW ON
  1. Ask Claude for the change (same as always)
  2. Claude hands back an updated zip
  3. Drag the folder onto Netlify Drop again
  4. Installed apps pick up the new version automatically
     (close and reopen twice if one seems stale)
  Team data lives in Supabase, so redeploys NEVER touch your data.

KNOWN LIMITS (honest list)
  - First-ever open on a device is slow (app compiles in the browser);
    after that it's cached and quick. A precompiled build would fix
    even the first open - ask Claude for the build setup when ready.
  - Chat/data sync is refresh-based, not instant push.
  - The anon key is visible to anyone with the URL: keep the URL
    internal. Proper per-user server auth is the next hardening step.
